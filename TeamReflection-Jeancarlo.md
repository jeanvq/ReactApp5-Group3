# Team Reflection — Jeancarlo Ricardo

## What went well during the team sprint?

The sprint went smoothly overall. Each team member had a clearly defined role from the start, which made it easy to work independently without stepping on each other's code. Gurinder set up the project scaffold and assigned tasks through the TEAM-GUIDE.md, so there was no confusion about who was building what. Having placeholder `div`s in `App.tsx` for each component was a smart move — it meant we could all work in parallel and slot our pieces in without conflicts.

## What would you do differently next time?

I would improve team communication. We mostly worked independently which was efficient, but checking in more regularly — even just a quick message to say "my part is ready" or "I ran into an issue" — would help the team stay more in sync. A short standup or a shared progress thread would make the integration phase smoother.

## What did you learn about working in a dev team?

I learned how to use GitHub in a more professional way. Creating a feature branch, working on it independently, and then opening a Pull Request for review felt like real industry workflow. It also made me realize how important it is to respect the structure someone else set up — you have to understand the project conventions (file locations, type definitions, naming) before writing a single line, otherwise your code won't integrate cleanly.

## How confident are you in contributing to a real team project?

I feel confident — nervous, but confident. This sprint showed me that I can take a defined task, understand the requirements, build it, test it, and deliver it through a proper Git workflow. The nerves come from knowing there is always more to learn, but the experience gave me a realistic and positive sense of what team development looks like.
